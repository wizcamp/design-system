# Layouts

> Stub — expand as patterns are established.

## Principles

- Pages have a clear nesting hierarchy: canvas → section → content. Aim for 2–3 visible levels.
- Each logical section of a page gets its own container. Do not combine unrelated content in one block.
- Page headers sit on the canvas, not inside a section container.

## Rules

- Use spacing tokens for all gaps and padding — never hardcode pixel values.
- Consult `tokens.md` for the correct background token at each layer.
- Do not use layout components (flex/grid wrappers) for visual styling — keep structure and appearance separate.
